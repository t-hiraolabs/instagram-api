# テンプレート・コラージュ機能の整理

**作成日**: 2026-07-09
**目的**: コード上に「テンプレート」「コラージュ」という言葉を使う機能が複数あり紛らわしいため、現状を棚卸しして整理する。**このドキュメントは現状把握のみが目的で、本ドキュメント作成時点ではコード変更は行っていない。**

---

## 1. 全体像（早見表）

| # | 機能 | 実体 | ユーザー向け表記 | DB | プラットフォーム |
|---|------|------|------------------|-----|------------------|
| 1 | 投稿文テンプレート | キャプション＋ハッシュタグの保存（端末内のみ） | 「テンプレートとして保存」「テンプレートから選ぶ」 | なし（SecureStore/localStorage） | ネイティブ＋Web |

## 2. 投稿文テンプレート（キャプション保存機能）

**実体**: `src/services/templateService.ts`（85行）

キャプション＋ハッシュタグ＋画像URLの組み合わせを、端末内だけに保存する機能。Supabaseには一切送信しない（コード内コメントで明記）。

```ts
interface PostTemplate {
  id: string; name: string; caption: string; hashtags: string[];
  type: 'feed' | 'story'; image_url?: string; created_at: string;
}
```

- 保存先: ネイティブは`expo-secure-store`、Webは`localStorage`。キーは`post_templates_v1`
- API: `getTemplates` / `saveTemplate` / `updateTemplate`（**未使用**）/ `deleteTemplate`
- UI: `ScheduleScreen.tsx`の投稿作成モーダル内
  - 「テンプレートとして保存」ボタン（1723〜1728行目）
  - 「テンプレートから選ぶ（N）」ボタン→ピッカーモーダル（1913〜1920行目、2356〜2401行目）

## 4. コラージュ機能

**実体**: `src/utils/createReel.ts`（507行）＋`src/components/CollageEditor.tsx`（354行）＋`src/utils/collageAssets.ts`（12行）

ファイル名は`createReel.ts`（リール作成）だが、中身はリールとは無関係で、写真グリッドのコラージュ合成エンジンになっている（歴史的な名残とみられる）。

- `CollageTheme`/`COLLAGE_THEMES`: 配色4種（ベージュ/ピンク/ミント/モノトーン）
- `CollageTemplate`/`COLLAGE_TEMPLATES`: レイアウトプリセット6種（simple1/stack2/grid4/polaroid/heroBottom2/circleBadge）
- `composeCollage()` / `composeTemplatePreview()`: 実際にCanvas 2D APIで描画する関数

`CollageEditor.tsx`は4ステップ（枚数→レイアウト→配色→編集）のウィザードで、レイアウト選択ステップの見出しが**「{枚数}枚のテンプレートを選ぶ」**（175行目）——Story Studioとは無関係な、3つ目の「テンプレート」という言葉の用法。

- 起動導線: `ScheduleScreen.tsx`の「コラージュ」ボタン（1255〜1258行目）→`openCollage()`
- DB: なし。完成画像はdata URLとして生成し、他の投稿と同じ画像アップロード経路に乗せる

### ⚠️ 確認済みの不具合: ネイティブアプリでクラッシュする可能性

`createReel.ts`は`document.createElement('canvas')`（422・486行目）を直接使っており、`Platform.OS`によるガードが一切ない。`CollageEditor.tsx`のレイアウト選択ステップは表示直後に`composeTemplatePreview()`（42・60行目）を呼んでプレビュー画像を生成するため、**iOS/Androidのネイティブアプリで「コラージュ」を開いてレイアウト選択画面に進むと、その時点で`document is not defined`のようなエラーが発生する可能性が高い**（実行環境の制約でこのセッションでは実機確認まではできていないが、コード上はWeb専用のAPIをガードなしで呼んでいることを確認済み）。

現状の起動ボタンはWeb/ネイティブ問わず表示されるため、ネイティブ利用者が踏むと壊れる導線が実在する可能性がある。対応要否は別途判断が必要。

## 5. フライヤー機能（未使用の死にコード）

**実体**: `src/utils/composeFlyer.ts`（162行）

写真上部55%＋下部に見出し/小見出し/箇条書き/価格バッジ/フッターを配置する「フライヤー」レイアウトのCanvas 2D合成エンジン。`FlyerSpec`/`FLYER_W`/`FLYER_H`/`drawFlyer()`/`composeFlyerImage()`を export しているが、**リポジトリ全体を検索してもどの画面・コンポーネントからも一切importされていない**。`composeStory.ts`のコメントに「FeedCropEditor / composeFlyer / composeFeedから共通利用されている」という記述はあるが、実際には配線されておらず、実装されたが使われなかった機能とみられる。

## 6. 気になる点まとめ

1. **命名の衝突**: `StoryTemplate`（Story Studio）・`PostTemplate`（投稿文保存）・`CollageTemplate`（コラージュのレイアウト）という3つの無関係な型が、コードを読む人には同じ概念に見えてしまう
2. **ユーザー向け文言の衝突**: 投稿文テンプレート機能とコラージュ機能の両方が、ユーザーに見える文言でも「テンプレート」という同じ言葉を別の意味で使っている（Story Studioは内部的に`templates`テーブルを使うが、ユーザー向け文言では「テンプレート」を使わずに回避している）
3. **死にコード**: `composeFlyer.ts`（162行、完全未使用）、`templateService.ts`の`updateTemplate`関数（未使用）
4. **ファイル名の実態とのズレ**: `createReel.ts`という名前だが中身はコラージュ合成エンジンで、リール機能とは無関係
5. **プラットフォーム安全性**: コラージュ機能がWeb専用のCanvas APIをガードなしで使っており、ネイティブアプリでクラッシュする可能性がある導線が存在する
6. **管理画面の非対称性**: `assets`テーブルには管理画面（`AdminAssetsScreen`）があるが、`templates`テーブル自体を管理するUIは存在せず、SQLでの直接操作が前提になっている

これらへの対応（削除・リネーム・修正など）は、必要になった時点で別途対応する。
