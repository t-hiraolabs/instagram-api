## 1. プロジェクト概要

### 1.1 サービス概要

**AImark（アイマーク）** は、「Instagram AIマーケティングエージェント」をコンセプトにした、日本の個人事業主・美容室・飲食店・サロン向けのInstagram運用支援アプリです。

単なる投稿作成ツールではなく、**AIチャットを入口として、分析→投稿企画→デザイン→キャプション→ハッシュタグ→投稿予約までを一気通貫で支援**することを目指しています。ホーム画面はそのままAIチャット画面になっており、ユーザーは「今日は何を投稿すればいい？」と聞くだけで投稿作成を始められます。

### 1.2 ターゲットユーザー

- 美容室、飲食店、ネイルサロン、整体院などの個人事業主
- SNS担当者のいない中小企業・フリーランス

### 1.3 技術スタック

| カテゴリ | 技術 |
|----------|------|
| モバイルフレームワーク | React Native 0.74 + Expo 51（Web版をVercelにデプロイ） |
| 言語 | TypeScript |
| 状態管理 | Zustand |
| バックエンド | Supabase（PostgreSQL + Deno Edge Functions + Storage + Auth） |
| AI | Anthropic Claude API（Sonnet / Haiku を用途別に使い分け） |
| SNS連携 | Instagram Graph API（Instagram Login） |
| 決済 | Stripe |
| Webホスティング | Vercel |
| E2Eテスト | Playwright（`@playwright/test`、`playwright.config.ts`で`expo start --web`をwebServerとして起動） |

## 3. 画面構成

| 画面（タブ） | ファイル | 概要 |

| 投稿（Post） | `ScheduleScreen.tsx` | 投稿作成・予約・下書き・カルーセル・タグ付け・「ストーリー作成」・AIチャットからの画像受け取りを担う中心画面 |

| ログイン | `AuthScreen.tsx` | メール/Google認証（`AccountBadge`内にモーダル表示） |

> **DM機能について**: `DMScreen.tsx`・`dmService.ts`は実装済みだが、Instagram側の`instagram_business_manage_messages`権限がMeta審査未承認で機能しないため、2026-07-16に下部タブ・ホームの導線を削除し、**UIから完全に非表示**にした（`RootNavigator.tsx`のタブ登録自体を削除）。コードは資産として残しており、権限承認後にタブを復元すれば再稼働できる。

`GenerateScreen.tsx` は現在チャット主導の設計に統合され、実質的にチャット（`ImageGenChat`コンポーネント）と`ScheduleScreen`が機能の中心になっています。手動でのストーリー作成（写真/動画に見出しを合成する機能）とリール作成機能は廃止済みで、`ReelScreen.tsx`・旧`StoryScreen.tsx`・`StoryEditor.tsx`は削除されています。

### 3.1 ストーリー作成（統合済み・移行中）

従来「コラージュ」（複数写真、Canvas合成）と「Story Studio」（写真1枚、AIおすすめ）に分かれていた2つの機能を、ユーザーには**「ストーリー作成」という1つの機能**として見せ、内部的にも共通のテンプレート型・共通の描画エンジンに統合した（2026-07-15〜16に実施）。

- **共通型**: `src/types/creativeTemplate.ts`の`CreativeTemplate`（`photoSlots` / `layers` / `textLayers`）。写真枚数は常に`photoSlots.length`から導出し、1枚のストーリーも複数写真のコラージュ型も区別なく同じ型で扱う。
- **共通描画エンジン**: `src/components/creative/CreativeCanvas.tsx`。描画順序（background→decorBehind→photo→decorFront→frame→text）を固定した単一レンダラーで、RN View + `react-native-gesture-handler` + `react-native-view-shot`方式（旧Story Studio方式）に統一。旧コラージュのCanvas 2D API（`document.createElement('canvas')`、Web専用でネイティブ未対応）は使わない。
- **写真スロットの指操作**: `DraggablePhotoSlot.tsx`が各スロットをクリップし、写真の位置・拡大率をpan/pinchで個別に調整できる（スロット2つ以上のテンプレートでは編集対象スロットの切替チップも表示）。
- **共有フォントプリセット**: `src/utils/fontPresets.ts`。`expo-font` + `@expo-google-fonts/*`で8書体をバンドルし、Web・ネイティブ両方で同じ見た目にした（旧Story Studioのフォント指定は実際には無効化されていた死んだ機能、旧CollageのGoogle Fonts動的注入はWeb専用だった）。
- **統合ギャラリー・エディタ**: `StoryGalleryScreen.tsx`（写真枚数・タグでの絞り込み付きテンプレート一覧）→`StoryTemplateEditor.tsx`（写真設定→位置/拡大率調整→テキスト編集→プレビュー→投稿へ）の一気通貫フロー。`ScheduleScreen.tsx`からはこの統合フロー1つだけが導線として表示される。
- **移行状況（DB）**: `templates`テーブルの`type`列は`'story'`/`'collage'`のまま未統一。`creativeTemplateService.listCreativeTemplates()`は現状`type IN ('story','collage')`で両方を横断的に読み、実行時に共通型へ変換して返す（DB側の`type`統一・RLSポリシー書き換え・生成カラム追加などは未実施。実データ0件のため影響は限定的）。
- **ロールバック用の温存コード**: `src/components/CollageEditor.tsx`（旧コラージュ）・`src/components/storyStudio/StoryStudioScreen.tsx`（旧Story Studio）は`ScheduleScreen.tsx`に引き続きマウントされているが、**タップできるUI導線は無い**（統合フローへの一本化が済んでいるため）。未使用コードの削除（旧統合計画のPhase 6/7）は未着手。
- **管理画面**: `AdminAssetsScreen.tsx`は引き続き旧`type='collage'`のテンプレートCRUD（背景画像・写真エリア・テキスト・装飾の配置、`TemplatePositionEditor.tsx`）を担当。新テンプレート管理UIへの統合も未着手。
- **回帰テスト**: `e2e/`配下にPlaywrightで、描画順序・スロット独立性・スロット切替の閾値・フィルタチップの回帰テストを整備済み。恒久のe2eハーネス画面は`src/screens/__e2e__/`（`?e2e=<name>`クエリで起動、ログイン不要）。

## 4. AIチャット（`ImageGenChat`コンポーネント）

### 4.1 位置づけ

ホーム画面に常時埋め込み表示（`embedded`モード）される他、`ScheduleScreen`からはモーダル表示でも呼び出せる共通コンポーネント。**Instagramマーケティングの評論家**として振る舞い、雑談には応じず、写真の撮り方・文章の書き方・投稿の仕方を厳しく分析してアドバイスすることに特化している。3〜10往復で結論が出るように設計。

### 4.2 主な機能

- **相談・評論**：業種・雰囲気などブランド情報を踏まえ、写真の構図・文章のフック・投稿頻度などを率直に評価し、具体的な改善アクションを提案する
- **自動分析**：「分析して」「プロフィールを見て」などのキーワードで、Instagramインサイト（フォロワー数・平均いいね/コメント・エンゲージメント率・投稿時間帯別反応・自己紹介文など）を自動取得し、**数値計算はプログラム側で行い、AIは説明・改善提案だけを担当**
- **会話履歴管理**：☰メニュー（ロゴタップで開閉）から会話の切替・名前変更・削除が可能。メニュー内にAI生成・チャットの利用量（回数ベース、送信直後に即時反映）も表示
- **アカウント別のチャット保存**：連携中のInstagramアカウント（最大3つ）ごとに会話を分けて保存。メニュー内のアカウントチップでワンタッチ切替
- **未送信の空会話は保存しない**、**会話は更新から30日で自動削除**（保持期間ポリシー）
- **未ログインユーザーはAI機能を一切利用できない**（`ensureLoggedIn`でガードし、アカウント作成を促すログイン/新規登録モーダルを表示）

### 4.3 AIコスト最適化設計

| 施策 | 内容 |
|---|---|
| モデルの使い分け | チャット・裏方分析（頻度高／重要度低）→ **Haiku**、投稿本体の生成（回数制限あり）→ **Sonnet**。クライアント指定は無視しサーバー側で強制 |
| プロンプトキャッシュ | 固定の指示文を`cache_control`でキャッシュし、可変部分のみ都度送信 |
| 履歴のトリミング | 直近16件（≒8往復）を超える古いやり取りは送らない |
| チャットの利用回数予算 | **メッセージ送信回数**で管理（下表）。旧仕様はトークン量ベースだったが、実際の課金コストとの相関が分かりにくかったため2026-07に回数ベースへ変更。UIには「残り◯回」として表示 |
| ブランド分析の裏上限 | 通常のAI生成回数を消費しない代わりに、別枠の裏上限で乱用を防止 |

## 6. データベース仕様（主要テーブル）

### profiles
プラン・AI利用回数・チャット利用回数・ブランド分析利用量・画像生成利用量・Stripe顧客IDなどを保持。列（抜粋）：`plan`, `ai_used`, `ai_period_start`, `chat_used`, `chat_period_start`, `img_used`, `img_period_start`, `brand_ai_used`, `brand_ai_period_start`, `assistant_memory`, `stripe_customer_id`。
> `chat_used`/`chat_period_start`は2026-07に「トークン量」から「メッセージ送信回数」へ意味が変わった（列名は変更なし）。移行時、Pro/ビジネスユーザーの残存トークン値が新しい回数上限と比較されて誤ブロックする不具合が発生したため、該当ユーザーの`chat_used`を0にリセットするマイグレーションを実施済み。

### scheduled_posts
投稿の予約・下書き・公開状態を管理。`type`（feed/story/reel）、`status`（draft/pending/published/failed）、`repeat`（くりかえし設定）、`user_tags`/`product_tags`/`location_id`（タグ付け）などを保持。

### brand_settings
Instagramアカウント（`ig_user_id`）ごとのブランド情報（業種・雰囲気・ターゲット層・トーン）。`unique(user_id, ig_user_id)`。

### chat_conversations / chat_messages
AIチャットの会話履歴。`chat_conversations`は`account_slot`（1/2/3）でアカウントごとに分離。`chat_messages`は`role`（user/assistant/image/user_image）と`content`を保持し、`ON DELETE CASCADE`で会話削除に連動。

### templates / template_tags / tags
「ストーリー作成」機能（旧コラージュ`type='collage'`・旧Story Studio`type='story'`）共通のテンプレートテーブル。`layer_defaults`（JSONB）にレイアウト・背景URL・写真エリア（`photoSlots`相当）・テキストレイヤー・装飾を保持。`tags`はギャラリー検索用（`templates.tags` text[]）とStory Studio由来のタグ一致スコアリング用（`template_tags`多対多）の両方で使われる。`type`列の`'story'`/`'collage'`統一は未実施（§3.1参照）。

### favorites / recent_templates / story_drafts
`favorites`はテンプレートのお気に入り機能だが、アプリコードから一切参照されていない（未使用）。`recent_templates`はStory Studio由来のAIおすすめが「直近使ったテンプレートを避ける」ためだけに使用。`story_drafts`は書き込み専用（保存はされるが読み返すUIが存在しない）。

### push_subscriptions
Web Push通知の購読情報。

### ig_first_analysis
Instagram連携直後の初回自動分析結果のキャッシュ。

> **2026-07-14に廃止**: `assets`/`asset_sheets`/`asset_tags`/`categories`テーブル（個別素材のスプライトシート管理）は、Story Studio・コラージュの両機能が「完成テンプレート方式」に統一されたことに伴い完全に削除された。管理画面の「素材シート管理」（シートアップロード・素材一覧タブ）も同時に廃止し、`AdminAssetsScreen.tsx`はコラージュの完成テンプレート管理のみを行う「テンプレート管理」画面になっている。

## 7. Edge Functions 仕様

### 7.1 `/claude`
Claude APIへの唯一の中継役。プランごとのAI生成回数・チャット利用回数・ブランド分析回数を`profiles`テーブルで管理し、リクエストの種別（`skipCount`＝ブランド分析等／`chat`＝チャット／デフォルト＝通常のAI生成）に応じて上限チェックとモデル振り分け（Sonnet/Haiku）を行う。3種別とも「フリー＝累計・リセットなし」「Pro/ビジネス＝月次リセット」で共通。デプロイ前には必ず`get_edge_function`で本番の現行コードを取得し、リポジトリのファイルとの差分がないか確認してから編集する運用にしている（過去に本番とリポジトリの内容が乖離していたため）。

### 7.2 `/instagram-insights`
Instagramプロフィール（フォロワー数・自己紹介文・リンク等）と直近投稿（いいね・コメント・リーチ）を取得し集計して返す。自動分析機能のデータソース。取得失敗時、Meta Graph APIが返すエラーメッセージ（`error`/`detail`）をほぼそのままクライアントに返しており、画面にはMeta側のエラー文言が生JSON付きで表示されることがある（アプリ側のバグではなくMeta側のアカウント設定・権限起因であることが多い）。

### 7.3 `/exchange-instagram-token`・`/publish-now`・`/publish-scheduled`
Instagram OAuthのトークン交換、即時投稿、予約投稿（毎分cron）を担当。

### 7.4 `/create-checkout`・`/stripe-webhook`
Stripeサブスクリプションの開始と決済結果反映。まだ本番（Live）モードへの切り替えは未実施（`docs/リリースチェックシート.md`参照）。

## 9. 「はじめてガイド」（オンボーディング）

`src/components/OnboardingChecklist.tsx`。新規ユーザー（全項目未達成の間）に表示される、順番達成型のチェックリスト。全項目達成すると非表示になり、代わりにマーケティングガイド（`MarketingGuideCard.tsx`）が表示される。

| # | 項目 | 達成条件 |
|---|---|---|
| 1 | AImarkアカウントを作成する | ログイン済み |
| 2 | Instagramと連携する | Instagram連携済み |
| 3 | ブランド設定を行う | ブランド名または業種が入力済み |
| 4 | AIに投稿ネタを相談してみる | チャット利用回数 > 0（フリープランの1回きりのお試しはこの導線からのみ許可） |
| 5 | 最初の投稿を作成する | 投稿が1件以上ある |

未達成の項目は、1つ前の項目が終わるまでロック（タップ不可・鍵アイコン表示）される。

> **既知の制限**: 4番目の達成判定は現在の期間のチャット使用回数（`chat_used > 0`）を見ているため、Pro/ビジネスユーザーはチャット利用回数が月次リセットされるたびに、達成済みでもガイドが未達成として再表示されてしまう不具合がある（フリープランは累計でリセットされないため影響なし）。恒久対応（永続フラグの追加）は未実装。

## 10. マーケティングガイド（`MarketingGuideCard.tsx`）

はじめてガイド完了後にホーム画面へ表示される、AIによるアカウント分析カード。

- **更新頻度**: フリープランは初回分析のみ（以降は自動更新されない）。Pro以上は毎週月曜0:00に自動更新（`getIsoWeekKey`でユーザーごとにキャッシュ）。
- **簡易チャット**: カード内に埋め込みのQ&Aチャット欄があり、分析結果について追加で質問できる。会話は永続化されず（ページを離れると消える）、チャットの利用回数を消費する。

## 13. 主要ユーザーフロー

```
[ログイン/アカウント作成] → [Instagram連携（OAuth）] → [ブランド設定] → [はじめてガイド]
   ↓
撮った写真やキャプション案をチャットに送る（はじめてガイド経由でフリープランでも1回だけ試せる）
   ↓
AIが評論家として構図・文章・投稿の仕方を評価し、改善アクションを提案
   ↓
投稿タブ（ScheduleScreen）でキャプション・写真を調整（プロンプトから生成／写真から生成／ストーリー作成）
   ↓
即時投稿 または 予約投稿
   ↓
（はじめてガイド完了後）マーケティングガイドが週次でアカウントを自動分析
```

分析フロー：
```
チャットで「分析して」→ プログラムが集計（保存率・いいね率・投稿時間帯等）
   ↓
AIが「何が良い/悪いか」「改善方法」を説明
   ↓
「この内容で投稿を作る」→ 上記の投稿作成フローに合流
```

## 14. 制限事項・今後の検討事項

1. Instagram Graph APIはビジネス/クリエイターアカウントが前提
2. 自己紹介文・プロフィールリンクはAPI仕様上取得できない場合があり、その場合は「未設定」としてAIに伝える設計
3. ハイライトの内容はAPIで取得不可（ユーザーからの手動共有が必要）
4. 画像生成AI（OpenAI）機能は削除済み（デザイン生成方式（Canvas/View合成）に一本化）
5. 管理ダッシュボード（MRR・利益率・解約率などの経営指標）は未着手
6. AI利用の不正利用対策（長時間セッション検知等）は基本的な月間上限のみで、詳細な仕組みは未実装
7. 旧コラージュ機能（`collageCompositor.ts`）は`document.createElement('canvas')`をガードなしで使っており、ネイティブアプリ（iOS/Android）で開くとクラッシュする可能性がある。ただし現在はUI導線が無いため実害は限定的（詳細は`docs/テンプレート・コラージュ機能整理.md`参照）
8. 「ストーリー作成」統合（§3.1）は Phase 1〜5 まで完了し本番導線として稼働中だが、DBの`type`列統一・旧コード（`CollageEditor.tsx`/`storyStudio/`）の削除は未着手
9. 「はじめてガイド」の4番目の項目（AIチャット相談）は、Pro/ビジネスのチャット利用回数が月次リセットされるたびに未達成表示へ戻ってしまう不具合がある（§9参照、恒久対応は未実装）
10. DM機能はMeta審査（`instagram_business_manage_messages`権限）未承認のためUIから非表示。コード自体は温存
11. Meta App Review（一般公開に必須）・特定商取引法に基づく表記・Stripe本番切り替えなど、本番リリースに向けたタスクは`docs/リリースチェックシート.md`にまとめている
