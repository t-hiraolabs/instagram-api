## 1. プロジェクト概要

### 1.1 サービス概要

**AImark（アイマーク）** は、「Instagram AIマーケティングエージェント」をコンセプトにした、日本の個人事業主・美容室・飲食店・サロン向けのInstagram運用支援アプリです。

単なる投稿作成ツールではなく、**AIチャットを入口として、分析→投稿企画→デザイン→キャプション→ハッシュタグ→投稿予約までを一気通貫で支援**することを目指しています。ホーム画面はそのままAIチャット画面になっており、ユーザーは「今日は何を投稿すればいい？」と聞くだけで投稿作成を始められます。

### 1.2 ターゲットユーザー

- 美容室、飲食店、ネイルサロン、整体院などの個人事業主
- SNS担当者のいない中小企業・フリーランス

### 1.3 技術スタック

| カテゴリ | 技術 |
|----------|------|
| モバイルフレームワーク | React Native 0.74 + Expo 51（Web版をVercelにデプロイ） |
| 言語 | TypeScript |
| 状態管理 | Zustand |
| バックエンド | Supabase（PostgreSQL + Deno Edge Functions + Storage + Auth） |
| AI | Anthropic Claude API（Sonnet 4.6 / Haiku 4.6 を用途別に使い分け） |
| SNS連携 | Instagram Graph API（Instagram Login） |
| 決済 | Stripe |
| Webホスティング | Vercel |

## 3. 画面構成

| 画面（タブ） | ファイル | 概要 |
|---|---|---|
| ホーム | `HomeScreen.tsx` | **画面全体がAIチャット**。上部にロゴ（会話履歴メニューの起点）。メッセージが無い間は今日のブリーフィング（ベスト投稿時間・おすすめネタ・TODO）を表示 |
| 投稿（Post） | `ScheduleScreen.tsx` | 投稿作成・予約・下書き・カルーセル・タグ付け・AIチャットからの画像受け取りを担う中心画面 |
| 分析 | `AnalyticsScreen.tsx` | インサイト分析（全プランで利用可） |
| DM | `DMScreen.tsx` | DM管理（近日対応） |
| プロフィール | `ProfileScreen.tsx` | アカウント連携・切替、ブランド設定、プラン管理 |
| ログイン | `AuthScreen.tsx` | メール/Google認証（`AccountBadge`内にモーダル表示） |

`GenerateScreen.tsx` は現在チャット主導の設計に統合され、実質的にチャット（`ImageGenChat`コンポーネント）と`ScheduleScreen`が機能の中心になっています。手動でのストーリー作成（写真/動画に見出しを合成する機能）とリール作成機能は廃止済みで、`ReelScreen.tsx`・旧`StoryScreen.tsx`・`StoryEditor.tsx`は削除されています。ストーリー投稿は「コラージュ」（Canvas合成）と「Story Studio」（レイヤー構造テンプレート＋AIおすすめ、`src/components/storyStudio/`）の2つの作成手段があり、どちらも完成画像を`type: 'story'`として投稿する形に統一されています。

### 3.1 Story Studio（`src/components/storyStudio/`）

写真を選ぶだけでInstagramストーリーを完成させる機能。テンプレートは画像ではなく
レイヤー構造（background→photo→frame→flower→decoration→text→cta）で管理し、
各レイヤーは`react-native-gesture-handler`+`reanimated`で指操作（移動・拡大縮小・回転）
できる。完成時は`react-native-view-shot`で1080x1920のPNGにキャプチャして書き出す
（コラージュ機能のCanvas方式とは異なり、Viewベースなのでネイティブアプリでも同じ
コードで動く）。

- DB: `templates`/`assets`/`categories`/`tags`/`template_tags`/`asset_tags`/`favorites`/
  `recent_templates`/`story_drafts`（`supabase/sql/22_story_studio.sql`）。タグは多対多。
  `templates.type`に`feed`/`carousel`/`reel_cover`も持たせてあり、将来Story以外にも対応できる設計。
- AIおすすめは2段階：①DB側でタグ一致数によるスコアリングで上位20件に絞り込み（無料）、
  ②Haikuにその20件の要約とブランド情報・投稿目的を渡し、最適な1件と差し替え素材・
  フォント・色をJSONのみで選ばせる（`recommendStoryTemplate` in `aiService.ts`）。画像生成は行わない。
- 編集内容は画像ではなくJSON（レイヤー配列）として`story_drafts.layers_json`に保存し、
  何度でも再編集できる。

## 4. AIチャット（`ImageGenChat`コンポーネント）

### 4.1 位置づけ

ホーム画面に常時埋め込み表示（`embedded`モード）される他、`ScheduleScreen`からはモーダル表示でも呼び出せる共通コンポーネント。**Instagramマーケティングの評論家**として振る舞い、雑談には応じず、写真の撮り方・文章の書き方・投稿の仕方を厳しく分析してアドバイスすることに特化している。3〜10往復で結論が出るように設計。

### 4.2 主な機能

- **相談・評論**：業種・雰囲気などブランド情報を踏まえ、写真の構図・文章のフック・投稿頻度などを率直に評価し、具体的な改善アクションを提案する
- **自動分析**：「分析して」「プロフィールを見て」などのキーワードで、Instagramインサイト（フォロワー数・平均いいね/コメント・エンゲージメント率・投稿時間帯別反応・自己紹介文など）を自動取得し、**数値計算はプログラム側で行い、AIは説明・改善提案だけを担当**
- **会話履歴管理**：☰メニュー（ロゴタップで開閉）から会話の切替・名前変更・削除が可能
- **アカウント別のチャット保存**：連携中のInstagramアカウント（最大3つ）ごとに会話を分けて保存。メニュー内のアカウントチップでワンタッチ切替
- **未送信の空会話は保存しない**、**会話は更新から30日で自動削除**（保持期間ポリシー）

### 4.3 AIコスト最適化設計

| 施策 | 内容 |
|---|---|
| モデルの使い分け | チャット・裏方分析（頻度高／重要度低）→ **Haiku**、投稿本体の生成（回数制限あり）→ **Sonnet**。クライアント指定は無視しサーバー側で強制 |
| プロンプトキャッシュ | 固定の指示文を`cache_control`でキャッシュし、可変部分のみ都度送信 |
| 履歴のトリミング | 直近16件（≒8往復）を超える古いやり取りは送らない |
| チャットのトークン予算 | 月間トークン数で管理（下表）。UIには「残り◯%」として表示し、利用制限を感じさせない設計 |
| ブランド分析の裏上限 | 通常のAI生成回数を消費しない代わりに、別枠の裏上限で乱用を防止 |

## 6. データベース仕様（主要テーブル）

### profiles
プラン・AI利用回数・チャットトークン使用量・ブランド分析利用量・Stripe顧客IDなどを保持。列（抜粋）：`plan`, `ai_used`, `ai_period_start`, `chat_used`, `chat_period_start`, `brand_ai_used`, `brand_ai_period_start`, `assistant_memory`, `stripe_customer_id`。

### scheduled_posts
投稿の予約・下書き・公開状態を管理。`type`（feed/story/reel）、`status`（draft/pending/published/failed）、`repeat`（くりかえし設定）、`user_tags`/`product_tags`/`location_id`（タグ付け）などを保持。

### brand_settings
Instagramアカウント（`ig_user_id`）ごとのブランド情報（業種・雰囲気・ターゲット層・トーン）。`unique(user_id, ig_user_id)`。

### chat_conversations / chat_messages
AIチャットの会話履歴。`chat_conversations`は`account_slot`（1/2）でアカウントごとに分離。`chat_messages`は`role`（user/assistant/image/user_image）と`content`を保持し、`ON DELETE CASCADE`で会話削除に連動。

### members
店舗スタッフ（ロスター表示用）。

## 7. Edge Functions 仕様

### 7.1 `/claude`
Claude APIへの唯一の中継役。プランごとのAI生成回数・チャットトークン・ブランド分析回数を`profiles`テーブルで管理し、リクエストの種別（通常生成／チャット／ブランド分析）に応じて上限チェックとモデル振り分け（Sonnet/Haiku）を行う。

### 7.2 `/instagram-insights`
Instagramプロフィール（フォロワー数・自己紹介文・リンク等）と直近投稿（いいね・コメント・リーチ）を取得し集計して返す。自動分析機能のデータソース。

### 7.3 `/exchange-instagram-token`・`/publish-now`・`/publish-scheduled`
Instagram OAuthのトークン交換、即時投稿、予約投稿（毎分cron）を担当。

### 7.4 `/create-checkout`・`/stripe-webhook`
Stripeサブスクリプションの開始と決済結果反映。

## 8. 認証・セキュリティ

| 項目 | 内容 |
|---|---|
| ユーザー認証 | Supabase Auth（メール／Google OAuth PKCE） |
| Instagram連携 | OAuth（Instagram Login）、スコープ: `instagram_business_basic`ほか |
| Row Level Security | 全テーブルに設定。自身のデータのみアクセス可 |
| Edge Function認証 | JWT検証 |
| APIキー保護 | Claude/OpenAI/StripeのシークレットキーはEdge Function内（サーバーサイド）のみ |
| レート制限 | `profiles`テーブル＋Edge Function側の二重管理 |

## 11. 主要ユーザーフロー

```
[ログイン] → [Instagram連携（OAuth）] → [ホーム画面（AIチャット）を開く]
   ↓
撮った写真やキャプション案をチャットに送る
   ↓
AIが評論家として構図・文章・投稿の仕方を評価し、改善アクションを提案
   ↓
投稿タブ（ScheduleScreen）でキャプション・写真を調整
   ↓
即時投稿 または 予約投稿
```

分析フロー：
```
チャットで「分析して」→ プログラムが集計（保存率・いいね率・投稿時間帯等）
   ↓
AIが「何が良い/悪いか」「改善方法」を説明
   ↓
「この内容で投稿を作る」→ 上記の投稿作成フローに合流
```

## 12. 制限事項・今後の検討事項

1. Instagram Graph APIはビジネス/クリエイターアカウントが前提
2. 自己紹介文・プロフィールリンクはAPI仕様上取得できない場合があり、その場合は「未設定」としてAIに伝える設計
3. ハイライトの内容はAPIで取得不可（ユーザーからの手動共有が必要）
4. 画像生成AI（OpenAI）機能は削除済み（デザイン生成方式（Canvas合成）に一本化）
5. 旧`StoryScreen.tsx`は未使用コードとして残存（削除候補）
6. 管理ダッシュボード（MRR・利益率・解約率などの経営指標）は未着手
7. AI利用の不正利用対策（長時間セッション検知等）は基本的な月間上限のみで、詳細な仕組みは未実装
