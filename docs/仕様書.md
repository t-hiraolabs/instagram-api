## 1. プロジェクト概要

### 1.1 サービス概要

**InstaAI** は、日本のInstagramユーザー（主に個人事業主・中小企業）を対象とした、AI支援によるInstagramコンテンツ生成・予約投稿管理モバイルアプリです。

Claude AIを活用してキャプション・ハッシュタグを自動生成し、Instagram Graph APIを通じて投稿・予約配信を行います。

### 1.2 ターゲットユーザー

- 美容サロン、カフェ、アパレルなどの個人事業主
- SNS担当者のいない中小企業
- Instagram運用を効率化したいビジネスオーナー

### 1.3 技術スタック

| カテゴリ | 技術 |
|----------|------|
| モバイルフレームワーク | React Native 0.74 + Expo 51 |
| 言語 | TypeScript 5.3 |
| 状態管理 | Zustand 4.5 |
| サーバーステート | TanStack Query 5.40 |
| バックエンド | Supabase (PostgreSQL + Deno Edge Functions) |
| AI | Anthropic Claude API |
| SNS連携 | Instagram Graph API v18.0 |
| Webホスティング | Vercel |

### 3.2 AI生成機能

`src/services/aiService.ts` で提供する機能:

| 関数 | 機能 | 出力 |
|------|------|------|
| `generatePost()` | テキストからキャプション+ハッシュタグ生成 | JSON |
| `generateStory()` | ストーリー専用コンテンツ生成 | JSON |
| `generateReelCaptions()` | リール用複数スライドキャプション | JSON |
| `generateFromImage()` | 写真からキャプション生成（Vision） | JSON |
| `generateFromImages()` | 複数写真からカルーセル用コンテンツ | JSON |
| `improveCaption()` | 既存キャプション改善提案（3案） | JSON |
| `refineCaption()` | 指示に基づくキャプション書き直し | string |
| `generateHashtags()` | 15〜20個の最適化ハッシュタグ生成 | string[] |
| `getSeasonalThemes()` | 月次季節イベントテーマ取得 | object |

#### AI生成パラメータ

- **業種 (industry)**: 美容、カフェ、アパレル、飲食店、ホテル、フィットネス など13種
- **トーン (tone)**: 明るい / プロフェッショナル / カジュアル / 感情的 / ユーモラス
- **季節テーマ**: お正月、バレンタイン、花見、夏祭り、クリスマス など12ヶ月分

### 3.3 投稿管理機能

#### 投稿ステータス

| ステータス | 説明 |
|----------|------|
| `draft` | 下書き（公開予定なし） |
| `pending` | 予約済み（公開待ち） |
| `published` | 公開済み |
| `failed` | 投稿失敗 |

#### 投稿タイプ

- `feed`: 通常フィード（単体・カルーセル）
- `story`: ストーリー（画像・動画）
- `reel`: リール（動画）

#### 繰り返し投稿オプション（Proプランのみ）

- `none`: 繰り返しなし
- `daily`: 毎日
- `weekly`: 毎週
- `monthly`: 毎月
- `weekdays`: 平日のみ

### 3.4 Instagram連携機能

`src/services/instagramService.ts` で提供する機能:

| 関数 | 機能 |
|------|------|
| `getMediaList()` | 最近の投稿一覧+分析データ取得 |
| `createMediaContainer()` | メディアコンテナ作成（2ステップ投稿の1step目） |
| `publishMedia()` | メディア公開（2ステップ投稿の2step目） |

## 5. Edge Functions 仕様

### 5.1 `/functions/claude`

Claude AIへのリクエストを中継するプロキシ。

**認証**: Bearer トークン (Supabase JWT)  
**レート制限**: DB の `profiles.ai_used` で管理  
**処理フロー**:
1. JWT検証
2. AI使用回数チェック（プラン上限と比較）
3. Anthropic APIへ転送
4. `profiles.ai_used` をインクリメント
5. レスポンス返却（上限時は 429 + アップグレード案内）

### 5.2 `/functions/exchange-instagram-token`

Instagram OAuth コールバックハンドラ。

**処理フロー**:
1. 認可コードを受け取る
2. 短期アクセストークンと交換
3. 60日間の長期トークンにアップグレード
4. プロフィール画像URLを取得
5. `{access_token, user_id, username, profile_picture_url}` を返却

### 5.3 `/functions/publish-now`

即時投稿処理。

**対応タイプ**: feed（単体・カルーセル）、story（画像・動画）、reel  
**処理フロー**:
1. メディアコンテナ作成
2. 処理完了をポーリング（`FINISHED` ステータス待機）
3. `media_publish` で公開
4. `{id, posted_type}` を返却

### 5.4 `/functions/publish-scheduled`

定期実行（cron）による予約投稿処理。

**実行間隔**: 毎分 (`* * * * *`)  
**処理フロー**:
1. `status='pending'` かつ `scheduled_at <= now()` の投稿を取得
2. 各投稿について `publish-now` を呼び出し
3. 結果に応じて `status` を `published` または `failed` に更新

## 6. 認証・セキュリティ

### 6.1 認証フロー

1. **ユーザー認証**: Supabase Auth（メール/パスワード）
2. **Instagram連携**: OAuth 2.0（Instagram Graph API）
   - スコープ: `instagram_business_basic`, `instagram_business_content_publish`, `instagram_business_manage_insights`
3. **トークン保管**: Expo SecureStore（ネイティブ）/ localStorage（Web）

### 6.2 セキュリティ設計

| 対策 | 実装 |
|------|------|
| Row Level Security | 全テーブルに設定（自身のデータのみアクセス可） |
| Edge Function認証 | JWT検証によるリクエスト認証 |
| APIキー保護 | Claude APIキーはサーバーサイドのみ使用 |
| レート制限 | DBトリガー + Edge Function で二重管理 |
| Supabase Service Role | Edge Function内でのみ使用（クライアントには非公開） |

## 11. ユーザーフロー

```
[新規登録/ログイン]
       ↓
[ブランド設定] (ProfileScreen)
  - ブランド名、業種、トーン設定
       ↓
[コンテンツ生成] (GenerateScreen)
  - 写真またはテキストから AI生成
  - 業種・トーン・季節テーマ選択
       ↓
[プレビュー・編集] (ScheduleScreen)
  - キャプション・ハッシュタグ編集
  - 投稿日時設定
       ↓
[Instagram連携] (ProfileScreen)
  - OAuth認証でビジネスアカウント接続
       ↓
[投稿] (即時 or 予約)
  - 即時: publish-now Edge Function 呼び出し
  - 予約: DB保存 → cron で自動投稿
```

## 12. 制限事項・注意点

1. **Instagram Graph API**: ビジネスアカウントまたはクリエイターアカウントが必要
2. **動画処理**: Instagramサーバー側の処理完了をポーリングで待機（数秒〜数分）
3. **ストーリー有効期限**: 24時間後に自動削除（Instagram仕様）
4. **カルーセル**: 複数画像は自動的に `CAROUSEL_ALBUM` として処理
5. **Web版制約**: SecureStore は未使用、localStorage で代替（セキュリティ上の考慮が必要）
6. **AI使用量**: 月次リセットは `ai_period_start` 基準で自動実行
